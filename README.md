# Fraud Detection MLOps Pipeline

End-to-end MLOps pipeline for detecting fraudulent financial transactions. Built on the PaySim synthetic dataset — covers ingestion, preprocessing, feature engineering, model training, experiment tracking, REST API serving, a Streamlit dashboard, and data drift monitoring, all wired together with Airflow.

---

## Results

| Metric | Value |
|--------|-------|
| Dataset | 88,213 transactions (PaySim) |
| Fraud rate | 9.31% |
| Features | 20 (11 preprocessed + 9 engineered) |
| Best model | XGBoost |
| Recall | 0.9957 |
| F1 Score | 0.9973 |
| ROC-AUC | 0.9995 |
| Tests | 31/31 passing |

> PaySim is a synthetic dataset with mathematically generated fraud patterns, which is why the scores are this high in it. Real-world performance on actual transaction data would be lower.

---

## Pipeline

```
prepare_dataset → load_to_mariadb → preprocess → engineer_features → train_models → mlflow_tracking
```

Each step is a standalone script under `src/` and is also wired as an Airflow task in the DAG. Redis and MariaDB are optional — every step falls back to CSV automatically if they're unavailable.

## Quick Start

### Run full pipeline

```bash
conda activate fraud_mlops

python src/ingestion/prepare_dataset.py
python src/ingestion/load_to_mariadb.py
python src/preprocessing/preprocess.py
python src/feature_engineering/engineer_features.py
python src/model_training/train_models.py
python src/mlflow_tracking/mlflow_integration.py
```

### Start the API and dashboard

```bash
# Terminal 1 — REST API
python src/api/fastapi_app.py
# → http://localhost:8000/docs

# Terminal 2 — Streamlit dashboard
streamlit run src/ui/streamlit_app.py
# → http://localhost:8501
```

### Docker

```bash
docker-compose up -d
```

### Run tests

```bash
# Full test suite (requires trained model files from running the pipeline)
pytest tests/ -v

# CI-safe subset (no model files needed)
pytest tests/test_pipeline.py -v
```

### MLflow UI

```bash
mlflow ui
# → http://localhost:5000
```

### Airflow

See `orchestration/README.md` for setup. Once running:

```bash
airflow dags trigger fraud_detection_pipeline
```

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/` | Root health check |
| GET | `/health` | Model load status |
| GET | `/model-info` | Best model metadata |
| POST | `/predict` | Single transaction prediction |
| POST | `/batch_predict` | Multiple transactions |
| GET | `/metrics` | Live prediction stats |

**Example:**

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{

    "type": "TRANSFER",
    "amount": 170136.0,
    "oldbalanceOrg": 170136.0,
    "newbalanceOrig": 0.0,
    "oldbalanceDest": 0.0,
    "newbalanceDest": 0.0
  }'
```
